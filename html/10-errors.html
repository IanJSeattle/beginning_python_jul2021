Errors in Python

Error messages in Python are (usually) very helpful.

You probably recognize this message (or something like it) already: 

Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list index out of range

This was created by trying to access element 5 of a four-element list.

Dissecting the Traceback

Here's what that means, in detail: 

Traceback (most recent call last):
"Hey, an error occurred. The bottom line is the most interesting one."

 File "<stdin>", line 1, in <module>
"We were in the file called <stdin>, on the first line, in the module called <module>."

<stdin> is a special filename you'll see when working in the interpreter directly.

<module> is used when there is no module name (more on modules later). 

IndexError: list index out of range

"This is the actual problem. I ran into an IndexError, and the message that came with it was 'list index out of range'."

What It All Means

You know the location of the error (<stdin>, line 1), and you know the type of error. You know the message that came with the error.

Look at the code indicated by the error: 

>>> my_list
['zero', 'one', 'two', 3]
>>> my_list[5]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
IndexError: list index out of range

My list index (5) is greater than the last index in the list (3), thus my error.

Errors Are Your Friend

Please take a moment to look at the error message when you have problems. They're usually very helpful, once you understand how they work.
