Programming Environment

A quick review of our Python programming environment

The command shell

This can be graphical, or text.

Text command shells are cmd.exe or PowerShell on Windows, or Terminal on Mac.

Graphical command shells are how you are used to working with the OS (point, click, drag, double-click).

The shell lets you run programs.

Text Command Shell

Primary uses:

  • Run programs
  • Handle stdin and stdout

Keeps track of which directory it's in

Useful to:

  • Run the Python interpreter
  • Start an editor (some OSes and editors)
  • Run Git commands if necessary

Running Python 

WASE-AM06002295:beginning_python_jan2020$ python3 hello_world.py
hello world!

This shows the prompt, then a dollar sign. The program to be run is python3. The argument to the program is hello_world.py, which Python interprets as a filename to be run.

Your command prompt will look different, depending on your OS.

Programs

We run programs on top of the OS, much like a car runs on a road. Programs we'll run in this class:

  • Python interpreter
  • Code editor
  • Web browser

Python Interpreter

A Python interactive session runs on top of the OS, launched from the command shell: 

>>> print('hello, world!')
hello, world!

Note that the prompt is different, compared to the command shell

Editor

Your code editor runs on top of the OS. IDEs like VSCode blur the line, because they can run a command prompt inside themselves.

Web Browser

Your web browser also runs on top of the OS.
