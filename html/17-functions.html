Functions in Python

Functions are the way to break up code into easily-reused "sections." Any code can go into a function, with some limitations. Functions take data in, and return data, doing some processing in the middle.

The most basic function: 

def say_hello():
    print('hello')

This is a silly example, but shows how you could encapsulate code within a function. To call it, you put say_hello() into your program.

More interesting: 

>>> def say_this(saying):
...   print(saying)
...
>>> say_this('hello')
hello
>>> say_this('and goodbye')
and goodbye

Now, it can be used to say anything.

Yet more interesting: 

>>> def say_multiple(saying, times):
...   for i in range(times):
...     print(saying)
...
>>> say_multiple('hey there', 3)
hey there
hey there
hey there

Arguments and Parameters

A parameter is the variable listed in the function definition (like saying in the previous example).

An argument is the data that's passed to that parameter (like 'hey there' in the previous example).

Return Values

Using the return keyword, you can return information from your function. Consider a simple addition function: 

def add(a, b):
    return a + b

This does a calculation (a + b), and uses return to send it back to the calling code: 

>>> my_value = add(3, 5)
>>> print(my_value)
8

Function Names

Name functions with verbs. They should do something. Names should be easy to read: verify_structure is better than vrfy_struct.

Function names allow your code to be much easier to read, with plain names rather than obscure collections of code.
